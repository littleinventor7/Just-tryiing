# 🚀 دليل رفع تطبيق Smart Flashcards & Quiz Generator على Vercel

تم إعداد المشروع وتجهيزه بالكامل ليكون متوافقاً مع منصة **Vercel** السحابية كـ **Monolithic serverless app**. 
إليك الخطوات التفصيلية لرفع موقعك وتشغيله مجاناً وبسهولة.

---

## 🛠️ ما تم تجهيزه وتعديله في الكود
1. **ملف الإعداد (`vercel.json`)**: تم إنشاؤه في المجلد الرئيسي ليوجه جميع طلبات الـ API (`/api/*`) إلى خادم Node.js الخلفي (`server.js`) بشكل تلقائي، بينما يتم تقديم ملفات الواجهة الأمامية (`public/`) كملفات ثابتة (Static Files) سريعة ومجانية مباشرة من شبكة Edge الخاصة بـ Vercel.
2. **تهيئة ملف التشغيل (`server.js`)**: تم تعديله وتصديره كـ Default Export متوافق مع Vercel Serverless Functions، مع تعطيل محاولة حجز الـ Port (`server.listen`) عند التشغيل داخل بيئة Vercel لتفادي أي أخطاء أثناء البدء (Startup Errors).

---

## 📋 الخطوات التفصيلية للرفع (Deployment Steps)

### الطريقة الأولى: الرفع المباشر عبر Vercel CLI (الأسرع والأسهل)

إذا كان لديك Vercel CLI مثبتاً على جهازك، يمكنك الرفع بـ 3 أوامر فقط:

1. افتح مبوب الـ Terminal في مجلد المشروع الرئيسي.
2. قم بتسجيل الدخول إلى حسابك في Vercel:
   ```bash
   vercel login
   ```
3. ابدأ عملية الرفع (سيقوم بإنشاء مشروع جديد وسؤالك عن الإعدادات الافتراضية، اضغط Enter للموافقة على الخيارات الافتراضية):
   ```bash
   vercel
   ```
4. بعد التأكد من أن النسخة التجريبية تعمل بشكل سليم، قم بالرفع النهائي للإنتاج (Production):
   ```bash
   vercel --prod
   ```

---

### الطريقة الثانية: الرفع عن طريق GitHub (الأفضل للتحديث المستمر)

هذه هي الطريقة الاحترافية التي تمكنك من تحديث الموقع تلقائياً بمجرد رفع أي تعديل على GitHub.

1. **إنشاء مستودع (Repository) على GitHub**:
   - قم بإنشاء مستودع جديد على GitHub (عام أو خاص).
   - قم برفع ملفات المشروع إليه باستخدام Git:
     ```bash
     git init
     git add .
     git commit -m "Initial commit for Vercel deployment"
     git branch -M main
     git remote add origin <رابط-المستودع-الخاص-بك>
     git push -u origin main
     ```

2. **الربط مع Vercel**:
   - اذهب إلى موقع [Vercel Dashboard](https://vercel.com/dashboard) وسجل دخولك باستخدام حساب GitHub.
   - اضغط على زر **"Add New"** ثم اختر **"Project"**.
   - ستظهر لك قائمة بمستودعات GitHub الخاصة بك، اختر مستودع المشروع واضغط **"Import"**.

3. **إعداد المتغيرات والرفع (Configure & Deploy)**:
   - في صفحة الإعدادات، **لا تقم بتعديل إعدادات Build and Development Settings** (الخيارات الافتراضية صحيحة بالكامل بفضل ملف `vercel.json` الذي قمنا بإنشائه).
   - (اختياري ولكنه مفضل جداً) أضف المتغيرات البيئية (Environment Variables) كما هو موضح أدناه.
   - اضغط على زر **"Deploy"**. سيستغرق الرفع أقل من دقيقة، وستحصل على رابط للموقع (مثال: `your-project.vercel.app`).

## 🔑 المتغيرات البيئية المدعومة (Environment Variables)

لجعل الموقع يعمل بالذكاء الاصطناعي مباشرة دون الحاجة لقيام كل مستخدم بإدخال مفتاح الـ API الخاص به في الإعدادات، يمكنك إضافة هذه المتغيرات في لوحة تحكم Vercel للمشروع (**Project Settings > Environment Variables**):

| اسم المتغير (Key) | القيمة (Value) | الوصف |
|-------------------|----------------|-------|
| `HACKCLUB_API_KEY` | `your_api_key` | مفتاح الـ API الافتراضي للذكاء الاصطناعي لتشغيل الموقع تلقائياً لجميع الزوار. |
| `GEMINI_API_KEY`   | `your_api_key` | بديل للمفتاح السابق (يعمل بنفس الكفاءة). |

## ⚠️ ملاحظة هامة جداً بخصوص حفظ البيانات (Data Persistence)

* **طبيعة Vercel Serverless**:
  منصة Vercel تعمل بنظام **Serverless Functions**، مما يعني أن الخادم الخلفي (`server.js`) يتم تشغيله وإيقافه بشكل ديناميكي ومستمر عند الطلب. 
* **حفظ بيانات المستخدمين**:
  نظراً لعدم وجود قرص تخزين دائم على Vercel، فإن أي مستخدم يقوم بالتسجيل أو حفظ حالته الدراسية من خلال ميزة الـ Login المدمجة بالموقع سيتم مسح حسابه أو بياناته عند إعادة تشغيل الـ Serverless Container (وهذا يحدث بشكل متكرر تلقائياً).
* **الحل الذكي المدمج**:
  التطبيق مصمم في الأساس ليعتمد بالكامل على **Local Storage** داخل متصفح المستخدم لحفظ الكلمات والدروس والتقدم الدراسي. وبالتالي، حتى لو تم مسح بيانات التسجيل من الخادم، **فلن يفقد الطالب أي تقدم دراسي** لأن المتصفح يحتفظ ببياناته محلياً وبشكل آمن تماماً ويعمل التطبيق بشكل طبيعي بدون تسجيل الدخول!
